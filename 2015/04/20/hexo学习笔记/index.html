hexo学习笔记

　　本文的目的是记录在用hexo搭建博客时遇到的一些问题和解决方法，因此，内容并非原创，都是针对特定问题
而搜索的解决方案。

打开GitBash输入hexo命令，遇到node: command not found

　　在安装好hexo和Nodejs第二天，打开打开GitBash输入hexo命令，遇到以下错误：

/C/Administrator/AppData/Roaming/npm/hexo: node: command not found
Node is not 'visible' in MINGW32 by default, only in the Windows Command Terminal. Same goes for npm

　　出现这种错的原因是Nodejs没有安装正确，环境变量没有设置。
　　参考链接分别是链接一和链接二。

hexo博客文章头部和侧边栏显示目录

实现概要

环境

  • hexo3.0；
  • 默认主题light

    修改或新建的文件

  • themes/light/layout/_partial/article.ejs
    在<%- item.content %>添加以下代码

    <% if (!index && item.toc){ %>
  <div id="toc" class="toc-article">
     <strong class="toc-title">文章目录</strong>
     <%- toc(item.content) %>
  </div>
<% } %>
<%- item.content %>

  • themes/light/layout/_widget/toc.ejs， 这是侧目录的widget
    在themes/light/layout/_widget目录下新建一个文件toc.ejs

     <h3 class="title"><%= __('Contents') %></h3>
   <div class="container" style="padding: 15px 20px;">
   </div>
</div><%- js('js/toc') %>

在这里需要注意的是要在themes/light/languages/zh-CN.yml 注明：Contents: 目录

  • themes/light/source/css/_partial/article.styl
    在themes/light/source/css/_partial/article.styl添加以下代码

    /*toc*/
.toc-article
background #eee
border 1px solid #bbb
border-radius 10px
margin 1.5em 0 0.3em 1.5em
padding 1.2em 1em 0 1em
max-width 28%
.toc-title
font-size 120%
#toc
line-height 1em
font-size 0.9em
float right
.toc
  padding 0
  margin 1em
  line-height 1.8em
  li
    list-style-type none
  .toc-child 
  margin-left 1em

  • themes/light/_config.yml
    在widgets的配置中增加toc

    widgets:

    • search
    • category
    • tag
    • toc
    • scrollup

  • themes/light/source/js/toc.js，代码见链接一

    部署后文章目录显示异常

    在hexo deploy后，本地的文章头部目录显示正常，而部署后博客文章目录显示不正常，这时需要进入.deploy-git文件夹，执行命令

    git push
